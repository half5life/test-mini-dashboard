# Челист проекта: Мини-дашборд заказов

Используйте этот чек-лист для отслеживания прогресса по каждому этапу. Для каждого нового этапа мы будем создавать отдельный чат.

## Этап 1: Подготовка инфраструктуры
- [x] Зарегистрировать демо-аккаунт в **RetailCRM**.
- [x] Получить API-ключ RetailCRM и базовый URL.
- [x] Создать бесплатный проект в **Supabase**.
- [x] Получить `Project URL` и `Service Role Key` (или `Anon Key`) от Supabase.
- [ ] Создать SQL-таблицу `orders` в Supabase.
- [x] Создать бота в **Telegram** через `@BotFather` и получить токен.
- [x] Узнать свой Telegram `CHAT_ID`.
- [x] Зарегистрироваться/авторизоваться в **Vercel** и **GitHub**.
- [x] Создать пустой GitHub репозиторий для проекта.

## Этап 2: Импорт в RetailCRM
- [ ] Инициализировать Node.js проект с TypeScript (`npm init`, `tsc --init`).
- [ ] Установить зависимости (`axios`, `dotenv`, `typescript`, `ts-node`).
- [ ] Создать файл `.env` и добавить ключи от RetailCRM.
- [ ] Написать TypeScript-скрипт чтения `mock_orders.json`.
- [ ] Реализовать маппинг полей JSON в формат API RetailCRM.
- [ ] Реализовать отправку POST-запросов к `/api/v5/orders/create`.
- [ ] Запустить скрипт и убедиться, что 50 заказов появились в интерфейсе RetailCRM.

## Этап 3: Синхронизация RetailCRM → Supabase
- [ ] Установить библиотеку `@supabase/supabase-js`.
- [ ] Добавить ключи Supabase в `.env`.
- [ ] Написать скрипт для получения списка заказов из RetailCRM (`/api/v5/orders`).
- [ ] Написать логику обработки данных (расчет итоговой суммы заказа, если нужно).
- [ ] Реализовать запись/обновление (upsert) полученных заказов в таблицу `orders` в Supabase.
- [ ] Запустить скрипт и проверить, что данные появились в таблице Supabase.

## Этап 4: Дашборд на Vercel
- [ ] Создать проект Next.js с TypeScript в корне репозитория.
- [ ] Настроить подключение к Supabase в Next.js (утилита `supabase-client`).
- [ ] Создать функцию для получения заказов из БД.
- [ ] Установить библиотеку графиков (например, `recharts`).
- [ ] Сверстать страницу дашборда с графиком (например, сумма заказов или кол-во заказов).
- [ ] Закоммитить и запушить код в GitHub.
- [ ] Настроить деплой в Vercel (привязать репозиторий).
- [ ] Прописать переменные окружения Supabase в настройках Vercel.
- [ ] Проверить работу дашборда по публичной ссылке Vercel.

## Этап 5: Telegram-уведомления
- [ ] Создать API Route в Next.js (например, `/api/webhooks/retailcrm`).
- [ ] Написать логику приема POST-запроса от RetailCRM.
- [ ] Написать логику проверки суммы заказа (> 50,000 ₸).
- [ ] Написать функцию отправки сообщения через Telegram Bot API.
- [ ] Задеплоить обновленный код на Vercel.
- [ ] Настроить триггер/вебхук в панели управления RetailCRM на URL нашего API Route.
- [ ] Сделать тестовый заказ на сумму > 50,000 ₸ в RetailCRM.
- [ ] Проверить получение уведомления в Telegram.
