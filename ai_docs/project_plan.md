# План реализации: Мини-дашборд заказов (TypeScript)

Этот документ содержит подробный план разработки проекта шаг за шагом. Проект будет реализован с использованием **TypeScript**, **Node.js** (для скриптов) и **Next.js** (для дашборда и API-вебхуков).

## Стек технологий
- **Язык:** TypeScript
- **Скрипты (Шаги 2 и 3):** Node.js (ts-node или компиляция tsc), axios/node-fetch, @supabase/supabase-js.
- **Frontend & Webhooks (Шаги 4 и 5):** Next.js (App Router), TailwindCSS (для базовых стилей), Recharts (для графиков).
- **База данных:** Supabase (PostgreSQL).
- **Хостинг:** Vercel.

---

## Этап 1: Подготовка инфраструктуры (Ручной этап)
На этом этапе создаются все необходимые учетные записи и подготавливаются ключи доступа (Credentials).
1. **RetailCRM:** Создать демо-аккаунт. Сгенерировать API-ключ с правами на чтение и запись заказов. Записать базовый URL (например, `https://your-domain.retailcrm.ru`).
2. **Supabase:** Создать новый проект. Создать таблицу `orders` с полями: `id` (uuid или int), `crm_order_id` (int/string), `total_sum` (numeric), `created_at` (timestamp), `status` (string). Получить `Project URL` и `Service Role Key` (или `Anon Key` для простых вставок).
3. **Telegram:** Через `@BotFather` создать бота, получить `BOT_TOKEN`. Узнать свой `CHAT_ID` (например, через `@userinfobot`), куда бот будет отправлять уведомления.
4. **GitHub & Vercel:** Создать пустой репозиторий на GitHub. Авторизоваться в Vercel через GitHub для будущего деплоя.

## Этап 2: Импорт моковых данных в RetailCRM
Разработка CLI-скрипта на TypeScript для разовой загрузки данных.
1. Инициализировать Node.js проект (`npm init -y`, `npm i typescript ts-node @types/node axios dotenv`).
2. Настроить `tsconfig.json`.
3. Создать файл `.env` для хранения `RETAILCRM_URL` и `RETAILCRM_API_KEY`.
4. Написать скрипт `scripts/import-to-crm.ts`:
   - Считать файл `mock_orders.json`.
   - Пройтись по массиву заказов.
   - Сформировать структуру данных, которую требует API RetailCRM (версия v5, метод `/api/v5/orders/create`).
   - Отправить данные пачками (batch) или по одному заказу (с небольшой задержкой, чтобы не упереться в лимиты API).

## Этап 3: Синхронизация RetailCRM → Supabase
Разработка скрипта для переноса данных из CRM в базу данных.
1. Добавить в `.env` ключи `SUPABASE_URL` и `SUPABASE_KEY`.
2. Написать скрипт `scripts/sync-to-supabase.ts`:
   - Сделать GET-запрос к API RetailCRM (`/api/v5/orders`).
   - Подсчитать общую сумму каждого заказа (исходя из количества и цены товаров), если CRM не отдает готовую сумму.
   - Преобразовать данные под формат таблицы `orders` в Supabase.
   - Использовать библиотеку `@supabase/supabase-js` для сохранения записей в БД (использовать `upsert`, чтобы избежать дубликатов при повторном запуске).

## Этап 4: Дашборд (Next.js) и Деплой на Vercel
Создание веб-интерфейса для визуализации данных.
1. Инициализировать проект Next.js (`npx create-next-app@latest dashboard --typescript`).
2. Настроить подключение к Supabase в Next.js (через переменные окружения).
3. Создать страницу, которая на сервере (Server Component) фетчит данные из Supabase.
4. Добавить библиотеку `recharts` и создать клиентский компонент (Client Component) графика (например, BarChart с распределением количества заказов по суммам или по датам).
5. Запушить код в GitHub.
6. Подключить репозиторий в Vercel, добавить переменные окружения и задеплоить проект.

## Этап 5: Telegram-бот и Уведомления
Реализация системы алертов о крупных заказах.
1. В Next.js проекте создать Route Handler (например, `app/api/webhook/crm/route.ts`).
2. Этот ендпоинт будет принимать POST-запросы.
3. В RetailCRM настроить триггер/вебхук: при создании заказа с суммой > 50 000 ₸ отправлять данные на наш Vercel URL (`https://your-app.vercel.app/api/webhook/crm`).
4. В коде Route Handler:
   - Принять webhook, извлечь сумму и детали заказа.
   - Если сумма > 50000, сделать HTTP-запрос к API Telegram (`https://api.telegram.org/bot<BOT_TOKEN>/sendMessage`) для отправки уведомления в нужный `CHAT_ID`.

## Этап 6: Финализация и сдача
Подготовка репозитория и отправка результатов.
1. Оформить `README.md` (добавить ссылки на Vercel и GitHub, скриншоты).
2. Описать в `README.md` процесс работы: какие промпты использовались, какие были трудности и как они решались.
